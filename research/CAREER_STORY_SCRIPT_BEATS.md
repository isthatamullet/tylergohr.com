# Tyler Gohr - Career Story Script (Beats Version)

## Overview
Interactive click/tap-to-advance presentation using accumulated text "beats."

**Format:** Reveal.js with fragments (text accumulates within each beat)
**Navigation:** Click/tap advances to next beat; text builds within beats
**Visual style:** Handwritten text effect, AI-generated imagery
**Target runtime:** ~2-2.5 minutes
**Target clicks:** ~25-30 total

---

## How Beats Work

```
[auto] Line appears automatically (1-2 second delay)
[CLICK] User clicks/taps to advance to next beat
```

Text accumulates on screen within each beat, giving users time to catch up before clicking.

---

## UX Principle

**All text accumulates and stays visible until next click.** Users control pace. Designed for partial-attention viewing (mobile, multitasking).

In 2026, 2-3 minutes = infinite. Users will glance away, check notifications, come back. Accumulated text means they never lose their place. Click points act as natural "checkpoints" - catch up, confirm ready, continue.

## Technical Notes for Reveal.js

### Configuration
```javascript
Reveal.configure({
  autoSlide: 150, // 150ms per word fragment (speaking pace)
  autoSlideStoppable: true,
  fragments: true,
  fragmentInURL: true // preserves state on back/forward
});
```

### Word-by-Word Animation (Native Fragments)
Use Reveal.js native fragments for smooth back/forth navigation. No external libraries needed.

```html
<!-- Example: One beat with word-by-word reveal -->
<section>
  <span class="fragment fade-in" data-fragment-index="1">It's</span>
  <span class="fragment fade-in" data-fragment-index="2">2008.</span>
  <span class="fragment fade-in" data-fragment-index="3">I'm</span>
  <span class="fragment fade-in" data-fragment-index="4">staring</span>
  <span class="fragment fade-in" data-fragment-index="5">at</span>
  <span class="fragment fade-in" data-fragment-index="6">47</span>
  <span class="fragment fade-in" data-fragment-index="7">browser</span>
  <span class="fragment fade-in" data-fragment-index="8">tabs.</span>
</section>
```

### Why Native Fragments (Not Typed.js or GSAP)
- **Smooth back/forth navigation** - Reveal.js optimized for this
- **No jitter on slide revisit** - fragments show/hide cleanly
- **No external dependencies** - reduces failure points
- **Mobile-tested** - tap navigation works perfectly

### CSS Transition for Smooth Feel
```css
.fragment.fade-in {
  opacity: 0;
  transition: opacity 0.15s ease-in;
}
.fragment.fade-in.visible {
  opacity: 1;
}
```

### Beat Structure
- Each beat = one `<section>` slide
- Words within beat = individual fragments with auto-advance
- End of beat = pause for user click/tap to next slide
- `autoSlide: 0` on final fragment of each beat to wait for click

## TEXT POSITIONING DESIGN DECISION

### Consistent Anchor Point
**Default position:** Left-center (left side of screen, vertically centered)
- First word of every slide starts at the same X,Y coordinate
- Additional lines stack below
- Reader always knows where to look

### Exception: Punchy Statements
Short emphatic lines can break pattern and go **centered**:
- "I'm building."
- "You're hired!"
- "That's the best part."
- etc.

**Why this works:** Center is still predictable (middle of screen). The pattern break becomes a SIGNAL - "this moment is different/important." Reader learns: text is usually left, BIG moments are center.

### CSS Implementation
```css
/* Default: left-center anchor */
.beat-content {
  position: absolute;
  left: 10%;
  top: 50%;
  transform: translateY(-50%);
  text-align: left;
}

/* Exception: centered emphasis */
.beat-content.centered {
  left: 50%;
  transform: translate(-50%, -50%);
  text-align: center;
}
```
